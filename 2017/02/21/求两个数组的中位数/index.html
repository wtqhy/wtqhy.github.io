LeetCode-1 求两个已排序数组的中位数

发表于 2017-02-21 |

题目分析

There are two sorted arrays nums1 and nums2 of size m and n respectively.

Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

Example 1:

Example 2:

题目的意思是，有两个已经排好序的数组nums1和nums2，其中nums1的长度为m, nums2的长度为n。让我们实现一个方法找出这两个数组的中位数，并且时间复杂度为O(log(m+n))。

看到这里，你心中有解决方案了吗？

解决方案

这说明，虽然先排序后，后查找中位数的方案很容易就想得到，而且即使不适用系统api，使用简单的快速排序，也能实现，但这显然不符合题目的要求。因此，我们不能使用先排序，再查找这种方法了。

方案二

那么让我们再捋一下该题的思路，要解决这个问题，我们首先要明白median（中位数）有什么用处，在统计数学中，中位数通常用来将一个集合set分割成两个长度相等的子集合subSet,并且其中一个子集合中的元素总是比另一的子集合的元素大。如果我们搞清楚了中位数的用途，那我们离答案就很近了。

首先，我们将nums1随机分成两部分：

因为nums1有m个元素，所以我们有m+1种分割方法，也就是i=0~m,并且我们知道，Len(left_nums1) = i, Len(right_nums1) = m - i。

注意： 当i=0时，left_nums1为空，当i=m，right_nums1为空。

同样的，我们也将nums2随机分成两部分：

然后，将left_nums1和left_nums2放入同一个集合，将right_sums1和right_sums2放入另外一个集合。如下图：

假如我们可以确保一下两个条件成立：

然后我们将sums1和sums2中元素分别放到left和right两部分中，并且使左右两个集合的长度相等，且右边集合的元素总是比左边集合的元素大，那么median = (max(left_part) + min(right_part))/2成立。

进一步分析

为了确保上面的两个条件成立，我们需要保证以下条件成立：

由上述条件一可知，因为i=0~m, 所以j=(m+n)/2-i或者j=(m+n+1)/2-i

  1. 简单起见，我们假设sums1[i-1],sums2[j-1], sums1[i],sums2[j]总是有效的数字，i=0,j=0,i=m,j=n这几种情形我们放在最后讨论。

  2. n必须大于等于m, 为什么呢？因为我们要确保j是非负数，而由条件0 <= i <= m和条件j = (m + n + 1)/2 - i可知，如果n<m，那么j就有可能为负数，所以需要n>=m。

Tips

n >= m的推导过程，我们假设i为最大，也即i = m：

所以最终方案为：

我们要在[0, m]区间内找到一个i，满足以下条件：

当j = (m + n + 1)/2 - i时，nums2[j-1] <=nums1[i]，nums1[i-1] <=nums2[j]成立。

我们可以按照以下步骤写一个二分查找的方法：

  1. 使imin = 0，imax = m，然后在区间[imin, imax]内搜索；
  2. 使i = (imin + imax) / 2，j=(m + n + 1) / 2 - i;

  3. 此时len(left_part)==len(right_part)成立，并且我们可能会遇到三种情形：

    a. 找到了i，使nums1[i-1] <= nums2[j],并且nums2[j-1] <= nums1[i] 成立，此时应该停止查找；

    b. 如果nums2[j-1] > nums1[i]，说明nums1[i]太小，我们应该增加i，以达到nums2[j-1] <= nums1[i]的状态。相应的，我们还要调整搜索区间，即imin = i + 1,然后跳转到第2步继续执行。

    讨论：

我们应该增加i，还是减小i，假设i增加, 那么相对的j会减小，那么nums1[i]就变大，nums2[j-1]就会变小，这样的话，就有机会达到`nums2[j-1] <= nums1[i]`的状态。

    c. 如果nums1[i-1] > nums2[j]，说明i过大，我们应该减小i，以达到nums1[i-1] <= nums2[j]的状态。相应的，我们还要调整搜索区间，即imax = i - 1,然后跳转到第2步继续执行。

当我们找到这样的i存在时，自然而然中位数也就很容易计算了：

  1. 如果m+n为奇数，那么median = max(nums1[i-1], nums2[j-1]);
  2. 如果m+n为偶数，那么median = (max(nums1[i-1], nums2[j-1]) + max(nums1[i], nums2[j]))/2;

现在我们需要考虑以下4种情形:

  1. 当i = 0时，nums1[i-1]不存在，也就是说nums1在left_part没有元素存在；
  2. 当i = m时， nums1[i]不存在，也就是说nums1在right_part没有元素存在；
  3. 当j = 0时，nums2[j-1]不存在，也就是说nums2在left_part没有元素存在；
  4. 当j = n时，nums2[j]不存在，也就是说nums2在right_part没有元素存在；

事实上这种4种情形是非常容易处理的。

我们所要做的是使max(left_part) <= min(right_part)成立，因此，如果i不等于0也不等于m，j不等于0也不等于n的话，那么nums1[i-1], nums2[j-1], nums1[i], nums2[j]都存在，这种情形下，我们必须检查nums2[j-1] <= nums1[i]和nums1[i-1] <= nums2[j]是否成立。

假如nums1[i-1], nums2[j-1], nums1[i], nums2[j]中的某个不存在的话，那么我们就不用检查nums2[j-1] <= nums1[i]或nums1[i-1] <= nums2[j]是否成立了。

举例

例1. 如果i = 0，那么nums1[i - 1]肯定不存在，此时nums1[i-1] <= nums2[j]一定成立，此时maxLeft = nums2[j-1];

例2. 如果i = m，那么nums1[i]肯定不存在，nums1[i-1] <= nums2[j]一定成立,此时minRight = nums2[j]。

以此类推即可得到所有的边界情况。

因此，我们需要考虑的是如何在[0, m]的区间内，当j = (m + n + 1)/2 - i时，需找到i，并做处理：

  1. 考虑i > 0 && nums1[i - 1] > nums2[j]，此时我们应该减小i;

  2. 考虑i < m && nums1[i] < nums2[j - 1],此时我们应该增加i;

  3. 找到i以后进行处理:

    • leftMax的取值情况:

      a. i = 0时, leftMax = nums2[j - 1];

      b. j = 0时, leftMax = nums1[i - 1];

      c. i处于(0, m)的开区间时，leftMax = max(nums1[i - 1], nums2[j - 1]);

    • rightMin的取值情况:

      a. i = m时, rightMin = nums2[j];

      b. j = n时, rightMin = nums1[i];

      c. i处于(0, m)的开区间时，rightMin = min(nums1[i], nums2[j]);

  4. 求median(中位数):

    • 如果m + n为奇数，median = leftMax
    • 如果m + n为偶数，median = (leftMax + rightMin) / 2

收尾

为什么是j = (m + n + 1) / 2 - i，而不是j = (m + n) / 2呢？

答：其实无所谓j = (m + n + 1) / 2 - i,还是j = (m + n) / 2，这完全是个人习惯问题，当m + n为奇数时，如果j = (m + n + 1) / 2 - i，halfLeft会比halfRight多一个元素，此时，中位数 = leftMax；如果j = (m + n) / 2 - i，halfLeft会比halfRight多一个元素，此时，中位数 = rightMin。
